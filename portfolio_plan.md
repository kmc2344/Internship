# 𠮷田尚翔 ポートフォリオ制作方針メモ

## 1. 制作目的

本ポートフォリオは、インターン面接時に提出することを目的とする。  
第一志望は **UI/UX系インターン** であり、特に以下の領域を中心に見せる。

- ユーザーリサーチ
- ヒアリング
- 体験設計
- アプリUIのプロトタイピング
- プラットフォームやサービスの体験設計
- 新規事業開発におけるUX的な関わり

単なる活動実績の羅列ではなく、  
**「人の声を聞き、課題を整理し、体験として成立する形に落とし込める人」**  
として伝わる構成にする。

## 21. PDF化を意識したCSS方針

Webページとして公開しつつ、印刷時にも崩れにくいようにする。

```css
@media print {
  body {
    background: #ffffff;
  }

  .section {
    break-inside: avoid;
    page-break-inside: avoid;
  }

  .print-break {
    page-break-before: always;
  }

  .no-print {
    display: none;
  }
}
```

### PDF化時の想定

ブラウザで印刷する場合：

- 用紙サイズ：A4
- 背景グラフィック：オン
- 余白：なし、または最小
- 倍率：100%
- ヘッダー・フッター：オフ

## 22. GitHub Pages公開方針

### 公開方法

1. GitHubでリポジトリを作成
2. `index.html`, `style.css`, `assets/` をアップロード
3. Settings → Pages
4. Branchを `main` に設定
5. `/root` から公開
6. 表示URLを確認

### 注意点

- 公開リポジトリにする場合、NDAに触れる内容を入れない
- ソニー共創の詳細、対象領域、成果物の具体情報は書かない
- プロトタイプ画面の画像を載せる場合は、公開してよいか事前確認する
- ダミー画像から実画像へ差し替える際も、公開可否を確認する
- 個人情報、連絡先、メールアドレスの掲載範囲に注意する
